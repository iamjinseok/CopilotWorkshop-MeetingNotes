# GitHub Copilot을 활용한 회의록 관리

GitHub Copilot과 GitHub Codespaces를 활용해 회의 메모를 정리된 회의록으로 변환하고, 누적된 회의록에서 팀의 할 일을 다시 찾아볼 수 있도록 구성한 저장소입니다. 사용자는 회의 메모를 Copilot Chat에 직접 입력하고, Copilot이 템플릿에 맞춰 정리한 결과를 `minutes/` 폴더에 저장해 활용할 수 있습니다.

## 주요 기능

- 자유로운 형식의 회의 메모를 표준 마크다운 회의록으로 정리
- 회의 내용에서 담당자, 작업 내용, 기한을 Action Item으로 추출
- 누적된 회의록을 기반으로 팀 전체 할 일 검색
- Copilot Agents 모드로 반복적인 회의록 정리 작업 자동화
- Copilot Ask 모드로 회의 히스토리와 할 일 현황 검색

## 사용 방법

### 1. Copilot Chat으로 회의록 만들기

회의 중 작성한 메모는 별도 파일로 저장하지 않고, Copilot Chat에 직접 입력하는 방식으로 사용합니다. 메모는 구조화되어 있지 않아도 되며, 참석자, 안건, 논의 내용, 할 일, 다음 일정 정도만 포함되어 있으면 Copilot이 충분히 정리할 수 있습니다.

예시 메모:

```text
[2026-04-03] 스마트 오피스 앱 개선 2차 기획 회의

참석자: 김팀장, 이철수(개발), 박영희(디자인), 최민수(기획)

장소: 제2회의실 및 온라인 병행

- 안건 1: 소셜 로그인 연동 이슈 및 보안 강화

철수님: 카카오/네이버 로그인 시 간헐적으로 토큰 만료 에러 발생하는 거 확인했음.
라이브러리 버전 업데이트가 필요할 것 같고, 보안 가이드라인에 맞춰서 OAuth 2.0 설정 다시 점검해야 함.
할 일: 철수님이 API 명세서 최신화하고 다음 주 수요일(4/8)까지 연동 테스트 완료하기로 함.

- 안건 2: 메인 화면 UI 컴포넌트 고도화

지난번 유저 테스트에서 폰트 가독성 지적 나옴.
영희님: 전체적으로 폰트 크기 1pt씩 키우고, 다크모드 대응해서 대비값 조정할 예정.
김팀장: 상단 배너 영역이 너무 큰 것 같으니 20% 정도 줄여서 콘텐츠가 더 많이 보이게 수정 부탁함.
할 일: 영희님이 수정된 디자인 시안(B안)을 이번 주 금요일(4/5)까지 피그마에 업로드하기.

- 안건 3: 상반기 베타 테스터 모집 계획

민수님: 4월 말에 오픈 베타 시작하려면 다음 주부터 테스터 모집 공고 나가야 함.
설문지 문항은 기획안에 있는 거 그대로 쓰되, 참여 보상(기프티콘) 예산 확인 필요.
김팀장: 예산은 승인 완료됐으니 진행하면 됨.
할 일: 민수님이 테스터 모집 공고문 초안이랑 설문지 링크 정리해서 다음 주 월요일(4/6) 오전까지 공유하기.

다음 회의는 4월 10일 오후 2시 예정.
```

Chat 창에 위와 같은 메모를 붙여넣은 뒤, 아래와 같이 요청하면 됩니다.

```text
위의 회의 메모를 #file:templates/meeting_template.md 양식에 맞춰 깔끔한 마크다운 회의록으로 정리해 줘.
누가(Owner), 언제까지(Deadline), 무엇을(Task) 해야 하는지 Action Item을 마크다운 체크박스로 명확히 작성해 줘.
```

Copilot이 생성한 결과를 검토한 뒤 `minutes/` 폴더에 저장합니다. 파일명은 `2026-04-03_스마트오피스앱기획회의.md`처럼 날짜와 회의명을 함께 쓰는 방식이 관리에 적합합니다.

### 2. Copilot Agents 모드로 회의록 추가 자동화하기

반복적으로 회의록을 정리해야 한다면 Copilot의 Agents 모드를 활용할 수 있습니다. 이 방식은 사용자가 매번 같은 프롬프트를 다시 작성하지 않고도, 회의 메모를 주면 동일한 규칙으로 회의록을 생성하도록 돕는 데 목적이 있습니다.

권장 방식은 다음과 같습니다.

- Agent가 사용자가 입력한 회의 메모를 받음
- Agent가 [templates/meeting_template.md](templates/meeting_template.md) 기준으로 회의록 형식을 맞춤
- Agent가 Action Item을 담당자, 작업 내용, 기한 기준으로 정리함
- 사용자는 결과를 검토한 뒤 [minutes](minutes) 폴더에 저장함

Agents 모드를 사용할 때는 다음과 같은 지시를 기준으로 구성하면 됩니다.

```text
사용자가 입력한 회의 메모를 분석해서 templates/meeting_template.md 양식에 맞는 회의록으로 정리한다.
Action Item은 담당자, 작업 내용, 기한이 빠지지 않도록 체크박스 형식으로 작성한다.
출력은 바로 minutes 폴더에 저장할 수 있는 완성된 마크다운 문서 형태로 제공한다.
```

### 3. Ask 모드에서 할 일 찾기

회의록이 `minutes/` 폴더에 누적되면 Copilot Ask 모드에서 필요한 정보를 다시 찾을 수 있습니다. 특정 회의 하나가 아니라 여러 문서를 한 번에 읽고 정리하는 용도로 적합합니다.

예를 들어 전체 할 일을 찾고 싶다면 다음과 같이 질문할 수 있습니다.

```text
@workspace minutes/ 폴더의 모든 회의록을 분석해서 팀원들의 Action Item만 담당자별 표 형태로 정리해 줘.
```

다른 활용 예시는 다음과 같습니다.

```text
@workspace 박영희의 현재 할 일만 모아서 보여 줘.
```

```text
@workspace 아직 완료되지 않은 Action Item만 찾아서 기한 순으로 정리해 줘.
```

```text
@workspace 소셜 로그인 또는 OAuth 관련 논의가 있었던 회의와 후속 작업을 찾아 줘.
```

이 방식으로 회의록 자체를 다시 읽지 않아도, 누가 무엇을 맡았는지 빠르게 확인할 수 있습니다.

## 폴더 구조

```text
.
├── templates/
│   └── meeting_template.md
├── minutes/
└── README.md
```

- `templates/`: 회의록 표준 양식을 보관하는 폴더
- `minutes/`: Copilot이 정리한 최종 회의록을 저장하는 폴더

## 운영 팁

- 회의 메모는 완전한 문장일 필요가 없고, 핵심 정보만 있어도 충분합니다.
- Action Item 추출 정확도를 높이려면 담당자와 기한 표현을 메모에 명확히 남기는 편이 좋습니다.
- 템플릿을 조직 표준에 맞게 바꾸고 싶다면 [templates/meeting_template.md](templates/meeting_template.md)를 먼저 조정한 뒤 같은 프롬프트를 계속 사용하면 됩니다.
- Ask 모드는 `minutes/` 폴더에 회의록이 누적될수록 더 유용해집니다.

## 시작 순서

### VS Code 또는 Codespaces에서 하는 방법

1. 저장소를 GitHub Codespaces 또는 VS Code에서 엽니다.
2. Copilot Chat을 열고 회의 메모를 직접 붙여넣습니다.
3. 템플릿 기준으로 회의록 정리를 요청합니다.
4. 결과를 검토한 뒤 `minutes/` 폴더에 저장합니다.
5. 이후 Ask 모드에서 누적된 회의록을 바탕으로 할 일을 검색합니다.

### GitHub Copilot 웹에서 하는 방법

1. GitHub에서 이 저장소를 연 뒤 Copilot Chat 또는 GitHub Copilot 웹 화면을 엽니다.
2. 회의 메모를 채팅창에 직접 붙여넣습니다.
3. [templates/meeting_template.md](templates/meeting_template.md) 기준으로 회의록 정리를 요청합니다.
4. 생성된 마크다운 결과를 복사해 저장소의 [minutes](minutes) 폴더에 새 파일로 저장합니다.
5. 이후 Copilot Ask 기능이나 저장소 검색을 이용해 누적된 회의록에서 할 일을 다시 찾습니다.
